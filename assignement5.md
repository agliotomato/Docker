## 1. 과제 목적
이번 과제는 FastAPI 애플리케이션을 Docker 환경에서 실행하고, 이를 AWS Learner Lab의 EC2 환경에 배포하는 실습입니다.
학생들은 이번 과제를 통해 다음 내용을 경험하게 됩니다.
- FastAPI 애플리케이션의 컨테이너화
- Dockerfile 작성
- docker run 또는 docker compose를 이용한 실행
- EC2 환경에서의 컨테이너 실행
- Docker의 재시작 정책(restart: always) 적용
- 브라우저를 통한 서비스 접속 확인
- GitHub와 데모 영상을 통한 결과 제출
## 2. 과제 내용
이번 과제에서는 FastAPI 단독 애플리케이션을 Docker로 실행하여 EC2에 배포합니다.이전에 수행한 courses 과제를 그대로 활용해도 됩니다.즉, 기존에 구현한 GET /courses, POST /courses 등의 FastAPI 앱을 기반으로
Docker 환경에서 실행 가능하도록 구성하면 됩니다.
핵심은 다음과 같습니다.
- FastAPI 앱을 Docker 컨테이너로 실행할 것
- 해당 컨테이너가 EC2 환경에서 동작할 것
- 외부에서 EC2 주소로 접속하여 FastAPI 서비스가 열리는 것을 확인할 수 있을 것
## 3. 구현 조건
**(1) FastAPI 단독 실행**

이번 과제는 FastAPI 단독으로 진행합니다.
Streamlit은 포함하지 않습니다.

**(2) Dockerfile 작성 필수**

프로젝트에는 반드시 Dockerfile 이 포함되어야 합니다.
즉, 학생은 FastAPI 앱을 Docker 이미지로 빌드할 수 있어야 하며,
실제로 해당 Dockerfile을 사용하여 컨테이너를 실행해야 합니다.

**(3) 실행 방식**

실행 방식은 아래 중 하나를 선택하면 됩니다.
docker run 명령어 사용
docker compose 사용
단, docker compose를 사용하는 경우 파일명은 docker-compose.yml 로 합니다.

**(4) Docker Hub 사용은 필수 아님**

이미지를 Docker Hub에 올리고 다시 docker pull 하는 방식은 필수가 아닙니다.
즉, EC2 내에서 직접 build하여 실행하면 됩니다.
## 4. 배포 환경
배포는 반드시 수업에서 제공된 AWS Learner Lab의 EC2 환경에서 수행해야 합니다.

포트 조건

- 외부 접속 포트는 반드시 80번 포트를 사용합니다.
- FastAPI 애플리케이션의 내부 포트는 자유롭게 설정해도 됩니다.

예를 들어 아래와 같은 방식이 가능합니다.

- 컨테이너 내부 FastAPI: 8000
- 외부 EC2 접속 포트: 80

즉, 외부에서는 브라우저로 http://EC2주소 형태로 접근 가능해야 합니다.
## 5. 재시작 정책 조건
이번 과제에서는 Docker 컨테이너에 restart: always 정책을 적용해야 합니다.

의미는 다음과 같습니다.

- Learner Lab 환경이 다시 켜졌을 때
- 조교 또는 교수자가 EC2에 접속했을 때
별도의 수동 실행 없이도 컨테이너가 자동으로 살아 있어야 합니다.

**중요**

데모 영상에서 자동 재시작 장면 자체를 시연할 필요는 없습니다.
다만, 실제 채점 과정에서 조교 및 교수자가 EC2에 접속했을 때 자동으로 서비스가 올라와 있어야 합니다.
## 6. 데이터 저장 관련
이번 과제에서는 볼륨(volume) 연결은 필수가 아닙니다.
즉, 기존 courses 과제의 JSON 파일 기반 구조를 그대로 활용하되, 컨테이너 재시작 이후 데이터 영속성까지는 요구하지 않습니다.
다만, 관심 있는 학생은 심화(advanced) 수준으로, Docker volume 연결까지 추가로 시도해 보아도 좋습니다.

## 8. 데모 영상 필수 포함 내용
데모 영상에는 반드시 아래 내용이 포함되어야 합니다.

**필수 포함 요소**

- EC2 터미널 화면
- Docker 컨테이너가 실행 중인 상태 확인
- docker ps 명령 결과
- 그 이후 브라우저에서 EC2 주소로 접속하는 장면
- FastAPI 서비스가 열리는 장면
- FastAPI 접속 방식

브라우저에서 FastAPI를 확인하는 방식은 아래 기준을 따릅니다.

브라우저에서 GET /courses 호출 시 JSON list가 바로 보인다면, /courses를 호출합니다.
만약 브라우저에서 해당 방식으로 확인하기 어렵다면, /docs 페이지를 호출해도 됩니다.

즉, 영상에서는 최소한 브라우저를 통해 FastAPI 서비스가 실제로 열리는 것이 확인되어야 합니다.
